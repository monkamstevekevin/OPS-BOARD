# Documentation Index

This folder documents the architecture, code structure, and the rationale (the “how” and the “why”) behind CAUREQ Ops Board. It is written to be friendly to new learners and practical for day‑to‑day contributors.

- architecture.md — big picture, data flow, key concepts
- backend.md — Spring Boot backend: layers, services, repositories, scheduling
- frontend.md — Angular frontend: components, services, routing, UX patterns
- apis.md — REST API reference (paths, params, responses)
- alerts.md — alerting model, thresholds, storage and UI
- missing-vm-detection.md — how we detect and handle VMs removed from Proxmox
- setup.md — environment, run, dev workflow, troubleshooting
- contributing.md — coding style, commit messages, how to propose changes

If you’re new, start with architecture.md, then setup.md, and browse backend.md or frontend.md depending on your focus.
